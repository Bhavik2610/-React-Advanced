=====================================================================
 TASK 5 — Unsubscribing from an onSnapshot listener on unmount
=====================================================================

------------------------- PROMPT USED --------------------------------

In a React component using Firestore's onSnapshot inside useEffect, what's
the correct cleanup pattern to unsubscribe the listener when the component
unmounts?

--------------------- ANSWER RECEIVED --------------------------------

onSnapshot RETURNS an unsubscribe function. Capture it, and return it (or
call it) from the useEffect cleanup:

useEffect(() => {
  const q = query(collection(db, "comments"), orderBy("createdAt", "desc"));

  const unsubscribe = onSnapshot(q, (snapshot) => {
    setComments(snapshot.docs.map((d) => ({ id: d.id, ...d.data() })));
  });

  // Cleanup: runs on unmount (and before the effect re-runs).
  return () => unsubscribe();
}, []);

------------------------- WHY IT MATTERS -----------------------------

- Without unsubscribe, the listener keeps running after the component is
  gone, trying to setState on an unmounted component (memory leak +
  "can't perform a React state update on an unmounted component" warning).
- With an empty dependency array [], the listener is created once and
  cleaned up once — exactly what we want for a live feed.

------------------------- INTEGRATION --------------------------------

Implemented in src/components/LiveCommentsFeed.jsx exactly as above.
The same pattern is also used in PlaylistEditor.jsx.
